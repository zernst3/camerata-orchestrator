# 2026-06-23 SSOT Emit Reconciliation: checks.toml replaces ci-checks.json

## Status

Implemented. `fix/ssot-emit-reconcile` branch.

## Problem

The apply/arm path (Layer 1 output) and the runtime checks path (Layer 2 + Layer 3
input) pointed at different files in different formats, so the governance apply loop
was never actually closed end-to-end.

### What the old arm/emit produced

`arm_files_for_repo` (in `crates/server/src/arm.rs`) emitted:

- `AGENTS.md` — prose rules
- `CONVENTIONS.md` — structured/mechanical rules
- `.camerata/rules.json` — gate config (rule id + chosen option)
- `.camerata/ci-checks.json` — JSON array of CI-tier rule metadata (id, title,
  directive, conformance)
- `.github/workflows/camerata-governance.yml` — a PLACEHOLDER scaffold that echoed
  each rule name with the message "wire each rule's enforcement manually"

### What the runtime actually consumed

- **Layer 2** (`camerata-checks` in-loop runner): reads `.camerata/checks.toml` via
  `crates/checks/src/manifest.rs` (`CheckManifest` / `ManifestCheck` / `load_manifest`).
- **Layer 3** (CI): `.github/workflows/camerata-gates.yml`, generated by
  `crates/server/src/workflow_gen.rs` (`generate_gates_workflow`).

### The gap

Neither `.camerata/ci-checks.json` nor `camerata-governance.yml` were consumed by
anything in the runtime. The SSOT manifest that Layer 2 and Layer 3 actually executed
(`.camerata/checks.toml`) was never emitted by the arm step. The placeholder workflow
was never a real CI gate; it printed rule names instead of running commands.

## Decision

Close the loop by making the arm/emit output exactly what the runtime consumes:

1. **Replace `.camerata/ci-checks.json` with `.camerata/checks.toml`** in the real
   manifest TOML format that `load_manifest` round-trips. Each applied CI-tier rule
   (mechanical or architectural) becomes one `[[check]]` entry using the real
   `ManifestCheck` struct: `id` = rule id, `name` = rule title, `command` = conformance
   hint when present (else a clearly-marked TODO placeholder), `severity = "high"`,
   `in_loop = true`. Tool-version-pinning fields are left absent; the architect adds
   them when wiring the real command.

2. **Replace the placeholder `camerata-governance.yml` with the REAL generated
   `camerata-gates.yml`** produced by `workflow_gen::generate_gates_workflow(&manifest,
   stack)` — the same function the `/api/projects/active/generate-ci-workflow` endpoint
   uses. This is the actual Layer-3 CI workflow, not a stub.

3. **Add `#[derive(Serialize)]` to `ManifestCheck` and `CheckManifest`** in
   `crates/checks/src/manifest.rs` so the arm path can build structs and serialize them
   via `toml::to_string`, guaranteeing the emitted file is identical in structure to what
   `load_manifest` expects. Round-trip fidelity is structural, not textual.

4. **Remove `.camerata/ci-checks.json`** from the emitted set. `.camerata/rules.json`
   (the armed-rule-id gate config) is kept; it is a separate concern from the executable
   check manifest.

## Files changed

| File | Change |
|------|--------|
| `crates/checks/src/manifest.rs` | Added `Serialize` to `ManifestCheck` and `CheckManifest` |
| `crates/server/src/arm.rs` | Replaced `ci_workflow()` + JSON emit with `emit_checks_toml_and_workflow()`; imports `CheckManifest`, `ManifestCheck`, `generate_gates_workflow`, `RepoStack`; updated all tests |
| `crates/server/src/onboard.rs` | Minimal test assertion update: `camerata-governance.yml` suffix -> `camerata-gates.yml` |
| `docs/decisions/2026-06-23_ssot_emit_reconciliation.md` | This file |

## Loop now closed

```
arm_files_for_repo()         produces   .camerata/checks.toml
                                               |
                   +-----------+----------+---+
                   |                          |
          Layer 2 (in-loop)           Layer 3 (CI)
     manifest::load_manifest()    generate_gates_workflow()
           ManifestCheckRunner        camerata-gates.yml
```

Both consumers read from the SAME manifest structs that the emit serialized. Any rule
an architect applies in the UI is immediately reflected in the Layer-2 dev-loop gate and
the Layer-3 CI backstop without a manual wiring step (the architect still fills in the
real command for TODO placeholders, but the manifest entry exists and is registered).

## Alternatives considered

- **Keep ci-checks.json as a parallel index.** Rejected: it served no runtime purpose;
  two files claiming authority over the same concern introduces drift.
- **Emit checks.toml via string templates rather than Serialize.** Rejected: textual
  template serialization cannot guarantee parse fidelity. Using the real structs means
  a serialization error is a compile-time or immediate runtime error, not a silent
  malformed-TOML that load_manifest silently rejects.
- **Use RepoStack::Rust as default for the emitted workflow.** Rejected in favor of
  `RepoStack::Unknown`: at arm time the stack may not be detected; the manifest checks
  are emitted regardless of stack, and the architect can re-generate with the detected
  stack via the dedicated endpoint.
